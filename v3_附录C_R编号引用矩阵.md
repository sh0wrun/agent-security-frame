## C.5 后续演进的材料追加机制

framework 的工程演进不可避免会引入新材料。新增材料按以下原则纳入引用矩阵:

**原则一**:新材料编号沿用 R 序列(R24, R25, ...),编号一旦分配不再回收。

**原则二**:新材料在第 2 章 §2.1 总览表登记,在所引用章节的"本章引用材料"段落补充,在本附录 C.1 矩阵补充对应行。

**原则三**:若新材料引发 framework 设计层面变化(如新增一个核心矩阵单元格、修正一条设计原则),需在前言或第 1 章 §1.5 补充版本演进说明,并把该材料加入 §C.2 "核心论据材料"列表。

**原则四**:若旧材料因学术撤回、工具下架、合规失效等原因不再可用,应在矩阵对应行追加"⊘"标注,但不直接删除——读者通过矩阵仍可还原 framework 当时的论据来源。

这四条原则保证 framework 的"论据可追溯性"——任何后续工程改动都不会让早期版本的论据链断裂。
