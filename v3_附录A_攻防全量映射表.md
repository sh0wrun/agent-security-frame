# 附录 A 攻防全量映射表

## A.4 4 层防御 × 防御机制清单

下表给出四层防御各自的核心防御机制,作为快速速查。

### 事前准入层

- **Skill 供应链扫描**:静态代码分析、依赖审计、签名链验证、行为模式画像
- **CVE 情报订阅**:GitHub Advisory Database 推送 + 触达响应
- **智能体注册中心**:统一白名单 + 权限发放
- **可重现构建验证**:字节级一致性核验
- **签名链可追溯**:发布者身份与签名核验
- **部署清单完整性**:从源码到构件的全链路记录
- **身份配置基线**:智能体与子智能体身份固化
- **知识基线**:系统提示词签名 + 知识库内容指纹

### 事中拦截层

- **预执行防火墙**:意图识别 + 策略匹配 + 异常分类 + 审计落库
- **统一策略引擎**:语义意图判决 + 词法绕过识别
- **能力凭证机制**:capability token 流转
- **沙箱与 Ring 隔离**:容器级与虚拟机级隔离
- **HITL 红线**:审批负载控制 + 上下文最小化 + 异常路径

### 事后运营层

- **防篡改审计链**:三档粒度(判决 / 会话 / 画像)+ WORM 存储
- **五维智能体画像**:Skill / 行为 / 权限 / 账号设备 / 模型上下文
- **Trust Decay 信任评分**:四级响应(绿 / 黄 / 橙 / 红)
- **持续合规约束**:不过夜审计 + 跨班次工单移交
- **显式汇报机制**:执行总结 + Skill 列表 + 资源清单 + 副作用清单

### 跨智能体层

- **去中心化身份**:W3C DID + 跨组织身份解析
- **智能体信任协议**:IATP 协同握手 + 会话级凭证
- **冒充检测**:密码学身份 + 行为指纹双层核验
- **多轮越狱检测**:会话级语义偏移度量
- **共享知识库防护**:内容指纹 + 来源标签 + 跨智能体变更追踪

## A.6 速查路径:从威胁到防御

工程团队在面对一个具体威胁时,可按以下三步速查:

**第一步**:在 §A.5 找到对应的 OWASP / MITRE / AIRT 编号,确认主防御层。

**第二步**:在 §A.1 与 §A.2 找到对应的攻击面与对抗技术,定位主战场所在章节。

**第三步**:在 §A.3 与 §A.4 找到对应的防御工具与防御机制,选择落地组合。

例如:面对"Memory Poisoning"威胁——第一步定位主防御层为事后(post/K);第二步定位攻击面为 S3 上下文窗口、对抗技术为 AT4 上下文操纵,主战场在第 6 章 §6.3;第三步定位防御机制为五维画像中的"模型上下文维度"+ 防篡改审计链 + Trust Decay 衰减,工具为 R5 AGT + AI-Sentinel 平台层。三步即可完成从威胁到落地组合的检索。

## A.7 涌现型攻击的事后检测路径

**背景**:涌现型攻击(以 R35 Viral Agent Loop 为代表,案例见第 5 章 §5.2.8)不落在 build-time 已知组件集合之上——第 3 章 §3.5.1 已明确:12 攻击面 S1-S12 是 build-time 完备的,涌现型攻击作为 runtime 补充而非替代,**不增设 S13、不扩张坐标系**。因此对其无法在事前准入侧静态枚举,只能在 runtime / 事后侧观测其扩散痕迹。

**检测路径**(均落在事后运营层,回链第 6 章 §6.3):

- **共享 KB / 长期记忆**:对写入分布做统计偏差告警与差异审计,捕捉异常的批量知识变更;
- **A2A 传播链**:对 agent 间消息附加来源标签(provenance),以污点追踪(taint tracking)还原传播路径;
- **跨 agent 知识变更**:对同一知识在多 agent 间的同步式异动做异常检测。

**诚实标注**:上述手段只把"未被察觉的扩散"压缩为"可观测的偏差",属于事后压缩而非根治;runtime 检测与事前准入之间的 cross/K 产业空白,详见第 6 章 §6.4.9。
